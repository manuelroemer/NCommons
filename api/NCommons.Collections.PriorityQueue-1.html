Remarks

The PriorityQueue<T> class is, at its heart, just a wrapper around another IPriorityQueue<T> implementation. This basically means that you don't have to use it at all - you could simply use any class that implements the IPriorityQueue<T> interface directly (for example the BinaryHeap<T>, which is also used by this class, if no other IPriorityQueue<T> instance was specified).

With that being said, using this class usually makes code more clear, because certain IPriorityQueue<T> implementations rename methods (for instance, the BinaryHeap<T> doesn't expose Enqueue / Dequeue methods; instead, they are called Push / Pop). It also makes understanding the instanciations easier. For instance, have a look at the following example:

    var pq1 = new BinaryHeap<int>();
    var pq2 = new PriorityQueue<int>();

Both result in an (implicit) IPriorityQueue<T>, but the second line makes that much more obvious.
