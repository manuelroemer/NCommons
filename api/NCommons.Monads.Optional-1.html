Class Optional<T>

A container which either holds a value of type T or not. See remarks for details.

Remarks

The Optional<T> type is very similar to .NET's Nullable<T> struct. In essence, it offers the same functionality: An Optional<T> instance can be empty (i.e. it does not hold/contain a value) or it can be non-empty (i.e. it holds/contains a value).

The major difference to the Nullable<T> type is that Optional<T> supports both value- and reference types, meaning that it can be used with both structures and classes. In addition, a held value of type T can be null which allows to model triple-state values. For example, an Optional<int?> may be:

  1. Empty, i.e. no value is present
  2. null
  3. A number, e.g. 123

The Optional<T> type should usually only be used if you require this triple-state. For dual state behavior, i.e. "Has a value" vs. "Doesn't have a value", null and Nullable<T> should still be used, especially with C# 8.0's support for nullable reference types.

As an example for when the previously mentioned triple-state is sensible, consider the following code fragment using the FirstOrDefault<TSource>(IEnumerable<TSource>) method:

var items = new[] { null, "Second", "Third" };
var first = items.FirstOrDefault();

if (first != null)
{
    Console.WriteLine("First item: {0}", first);
}
else
{
    Console.WriteLine("There were no items.");
}

// Output:
// "There were no items."

The code above displays a simplified, but common usage of FirstOrDefault<TSource>(IEnumerable<TSource>). If the result is the default value (here: null), users often assume that a list is empty. This is obviously not true for the list above - the first value was simply the default value of type String.

This problem could be solved with an optional. By returning an Optional<T>, the method could return no value (i.e. an empty optional), the default value (i.e. null) or any other string. The following example shows how the extension may look like, using an optional:

static Optional<string> BetterFirstOrDefault(this string[] items)
{
    if (items.Length == 0) return Optional<string>.Empty;
    return new Optional<string>(items[0]);
}

// Possible usage:
var items = new[] { null, "Second", "Third" };
var first = items.BetterFirstOrDefault();

if (first.HasValue)
{
    Console.WriteLine("First item: {0}", first.GetValue() ?? "null");
}
else
{
    Console.WriteLine("There were no items.");
}

// Output:
// "First item: null"

In addition to the members defined in this class, the Optional class defines additional utility functions which further enhance the interaction with the Optional<T> type.

Remarks

The HasValue property can be used to determine whether the optional holds a value. For example, the following code segment uses the HasValue property to determine whether GetValue() can safely be called.

Optional<int> opt = 123;

{
    Console.WriteLine("The value is: {0}", opt.GetValue());

Two optionals are considered equal if:

  • They are both empty.
  • They are both non-empty and their two held values are considered equal by a default EqualityComparer<T> instance.

If the above conditions are not met, the two optionals are considered to be unequal.

FlatMap<TResult>(Func<T, Optional<TResult>>)

If the optional is non-empty, the specified flatMapValue function is invoked with the optional's held value. The returned value is then returned by this method.

If the optional is empty, the flatMapValue function is not invoked. Instead, an empty Optional<T> instance is returned.

Declaration

public Optional<TResult> FlatMap<TResult>(Func<T, Optional<TResult>> flatMapValue)

Parameters

Func<T, Optional<TResult>> flatMapValue

The function to be invoked if the optional is non-empty. This function receives the optional's held value as a parameter.

Returns

Optional<TResult>

The Optional<T> instance returned by the flatMapValue function if the optional is non-empty or an empty Optional<T> instance if the optional is empty.

Type Parameters

TResult

The type of the result which gets returned by the flatMapValue function.

Remarks

FlatMap<TResult>(Func<T, Optional<TResult>>) allows you to transform the held value of a non-empty optional. Calling the method on an empty optional has no effect.

The following code segment shows how map can be used to transform the value of an Int32 optional to a String optional:

Optional<int> intOpt = 123;
Optional<string> stringOpt = intOpt.FlatMap(value => new Optional<string>(value.ToString()));
Console.WriteLine(stringOpt.GetValue());

// Output:
// "123"

In addition to the FlatMap<TResult>(Func<T, Optional<TResult>>) function, there is also a function called Map<TResult>(Func<T, TResult>). The difference between the two functions is how return values of type Optional<T> are handled. In essence, FlatMap<TResult>(Func<T, Optional<TResult>>) doesn't wrap a returned Optional<T> into another Optional<T> instance, while Map<TResult>(Func<T, TResult>) does exactly that. The following code segment demonstrates this behavior:

Optional<int> opt = 123;
Optional<Optional<string>> map = opt.Map(value => new Optional<string>(value.ToString()));
Optional<string> flatMap = opt.FlatMap(value => new Optional<string>(value.ToString()));

As you can see, FlatMap<TResult>(Func<T, Optional<TResult>>) flattens an optional which is returned by the mapper function, while Map<TResult>(Func<T, TResult>) does not.

GetValue()

Returns the held value if the optional is non-empty and throws an exception for an empty optional.

Declaration

public T GetValue()

Returns

T

The held value if the optional holds a value.

Remarks

The GetValue() function should be used if you either know that an optional holds a value or if you must ensure that an optional is not empty.

The following code segment first checks if an optional holds a value by checking the HasValue property. Afterwards, it prints a text depending on that value.

The following code segment shows under which circumstances GetValue() throws an exception:

Optional<int> nonEmptyOptional = 123;
Optional<int> emptyOptional = new Optional<int>;

Console.WriteLine(nonEmptyOptional.GetValue());
Console.WriteLine(emptyOptional.GetValue());

// Output:
// "123"
// InvalidOperationException

GetValue() is most useful when you must ensure that an optional holds a value before your code continues.

As an example, the following code segment executes a critical piece of business logic which absolutely requires an instance of type RequiredValues. By calling GetValue() before invoking the business logic, the code ensures that the process doesn't start if the required values are missing:

Optional<RequiredValues> opt = GetRequiredValues();

// Throws if the optional is empty, i.e. if it doesn't hold a "RequiredValues" instance.
RequiredValues requiredValues = opt.GetValue(); 

RunCriticalBusinessLogic(requiredValues);

If computing the substitute parameter is a heavy operation, it is recommended to use the GetValueOr(Func<T>) overload which lazily creates the substitute value.

Optional<string> emptyStringOpt = Optional<string>.Empty;

Console.WriteLine(nonEmptyStringOpt.GetValueOr("Substitute value"));

// Output:
// "Substitute value"
// "Hello"

Another possible use case for this method is throwing custom exceptions if an optional doesn't hold a value:

Remarks

Match(Action<T>, Action) is a utility function which allows you to run two different pieces of code, depending on whether an optional holds a value or not.

The following code segment shows how Match(Action<T>, Action) can be used to print an optional's state to the console.

static void Print(Optional<string> opt) =>
    opt.Match(
        value => Console.WriteLine("Value: {0}", value),
        ()    => Console.WriteLine("Empty")
    );

Optional<string> emptyOpt = Optional<string>.Empty;
Optional<string> nonEmptyOpt = "Hello";

Print(emptyOpt);
Print(nonEmptyOpt);

// Output:
// "Empty"
// "Value: Hello"

ToVariant()

Returns a new Variant<T1> which represents the same state as the optional, i.e. it holds a value if the optional is non-empty or it is empty if the optional is empty.

Declaration

public Variant<T> ToVariant()

Returns

Variant<T>

A new Variant<T1> holding the optional's held value if the optional is non-empty or an empty Variant<T1> if the optional is empty.

Remarks

The Variant<T1> essentially has the same functionality as an Optional<T>. Both types either hold a value or are empty. Therefore, any Optional<T> can easily be converted to a Variant<T1> and vice versa.

Nevertheless, the two types serve different purposes. While an Optional<T> is mainly designed for modeling the absence of a value, the variant family is designed for scenarios where one needs to accept or provide an instance of one of many types. Depending on the use case, you may pick either type for your scenario. If you ever need to switch between the two, you can use one of the available conversion methods.
